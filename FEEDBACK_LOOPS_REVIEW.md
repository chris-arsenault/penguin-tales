# Feedback Loops Review (Default Project)

Reviewed the default project configs under `apps/canonry/webui/public/default-project` with all systems/actions/generators assumed enabled.

## Potential Broken Loops
- `war_outbreak` requires the `war_brewing` tag, but no system sets it (war_brewing_detector only sets `conflict`/`hostile`), so the war template likely never fires. (`apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/systems.json`)
- Death-based chains are inert: nothing sets NPCs to `dead`, so `power_vacuum_detector`/`legend_ripe_detector` never trigger, which blocks `succession_crisis` and `legend_crystallization`. (`apps/canonry/webui/public/default-project/actions.json`, `apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/systems.json`)
- Recovery/migration tags are filtered but never assigned (`recovery_ready`, `prosperity_ready`, `thaw_candidate`, `extreme_hot`, `extreme_cold`, `temperate`), so thermal/economic recovery, ice memory thaw, and thermal migration won’t fire. (`apps/canonry/webui/public/default-project/generators.json`)
- `guild_trade_expansion` and `cult_mystical_influence` filter on `guild`/`cult` tags, but `guild_establishment`/`cult_formation` only set subtypes and other tags; unless subtypes auto-tag, these systems never trigger. (`apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/generators.json`)
- `kill_orca` has no mutations (no status/tag/pressure changes), so orca threats can’t be reduced through actions. (`apps/canonry/webui/public/default-project/actions.json`)
- `location_discovery` creates `resource_node`/`anomaly` without `resource`/`anomaly` tags, so `resource_diffusion`/`thermal_diffusion` may ignore those discoveries unless subtype auto-tags. (`apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/systems.json`)

## Runaway / Side-Effect Risks
- Harmony spiral: war/raid/draw/escalate actions reduce harmony and are pressure‑modulated; colony_founding adds `leader` tags counted as negative harmony, which can keep harmony low and favor repeated conflict. (`apps/canonry/webui/public/default-project/actions.json`, `apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/pressures.json`)
- Security coupling: `hostile` tags from war_brewing_detector and orca_raider_hostility are counted against security; low security triggers orca_raider_arrival, adding more hostile/raider/orca tags. (`apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/pressures.json`)
- Magical stability extremes: corruption actions push stability down with pressure modifiers; `mystical`/`wild`/`anomaly` tags from cults, discoveries, and cultural drift also count negative, which can trap stability low. (`apps/canonry/webui/public/default-project/actions.json`, `apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/pressures.json`)
- Resource oscillation: establish_trade and economic_prosperity_detector increase resource_availibility while economic_stress_detector decreases it when `stressed` tags appear; combined with resource_diffusion, this can swing or ratchet. (`apps/canonry/webui/public/default-project/actions.json`, `apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/pressures.json`)
- Tag/status mismatch: thermal_recovery_detector sets `thriving` tag without changing status; several generators use status filters, so “waning status + thriving tag” can cause odd triggers. (`apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/generators.json`)
- Conflict-network amplification: conflict_contagion spreads `enemy_of` via `allied_with`/`member_of` while alliance_formation creates allies from shared enemies, densifying conflict graphs quickly. (`apps/canonry/webui/public/default-project/systems.json`)

## Feedback Loops / Interactions
- Security ↔ orca/hero loop: low security spawns orcas, hostile tags reduce security; hero_emergence and crisis_legislation push it back up. (`apps/canonry/webui/public/default-project/pressures.json`, `apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/systems.json`)
- Harmony ↔ war/alliance loop: war actions reduce harmony; conflict_contagion spreads enemy relationships; alliance_formation and great_festival raise harmony. (`apps/canonry/webui/public/default-project/pressures.json`, `apps/canonry/webui/public/default-project/actions.json`, `apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/generators.json`)
- Resource ↔ economy loop: discovery/bloom add resources; colony_founding adds colonies (resource drag); diffusion tags thriving/stressed; economic stress/prosperity modifies resource_availibility and harmony/security. (`apps/canonry/webui/public/default-project/pressures.json`, `apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/actions.json`)
- Magic ↔ anomaly/cult loop: magic discovery and wild magic increase stability but add mystical/wild tags; cults and corruption actions reduce stability; anomaly discovery lowers stability further. (`apps/canonry/webui/public/default-project/pressures.json`, `apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/actions.json`, `apps/canonry/webui/public/default-project/systems.json`)
- Ideology ↔ governance loop: ideology_emergence creates rules; belief_contagion spreads them and boosts harmony/security; legal_code_formation clusters rules and raises security. (`apps/canonry/webui/public/default-project/generators.json`, `apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/pressures.json`)
- Thermal ↔ stress/recovery loop: thermal_diffusion tags safe/dangerous; thermal_stress_detector and thermal_recovery_detector add stressed/thriving/emergent tags; decline/recovery generators change status. (`apps/canonry/webui/public/default-project/systems.json`, `apps/canonry/webui/public/default-project/generators.json`)

Eras modulate template weights/system multipliers and add entry/exit pressure deltas, so the above loops shift intensity by era (conflict boosts conflict contagion, innovation boosts clustering, invasion boosts orca activity). (`apps/canonry/webui/public/default-project/eras.json`)

Assumed subtype does not auto‑apply matching tags; if it does, the guild/cult/resource/anomaly tag issues may be less severe. No code/config changes made.
